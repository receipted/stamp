## THE BIG PICTURE

Ben is building a **semantic governance substrate** — a system where stochastic AI output is abundant but human authority remains explicit, durable, and inspectable. The product name has evolved: Surface → HYP → Card Protocol. The core thesis:

> "The system may prepare indefinitely. It may never decide without a remembered human action."

The substrate is NOT an app. It's a **protocol layer** that sits between raw AI output and governed meaning. Think of it as an epistemic operating system: immutable history + lazy evaluation + human-gated reduction.

---

## LAYER MAP (what the substrate actually is)

### The Pipeline: `Record → Shape → Surface → Collab`

1. **Ore** — Raw unshaped material (chat transcripts, notes, code, voice). Captured with provenance.
2. **Sieve** — Governed compression. Separates signal from noise, declares what was lost. Pure function: same input + same policy = same output.
3. **Graph** — Argument topology. Claims as nodes, support/attack/context as edges. Graph-based relevance replaces keyword matching.
4. **Type** — Semantic frame system. Claims typed by persistence role: load-bearing, scaffolding, contested, superseded, bridging. Prevents false collapse across models.
5. **Spec** — Falsifiable commitment with provenance. Not documentation — a versioned belief with invalidation conditions.
6. **Check** — Universal verification. Takes any artifact + contract, returns structured pass/fail. Enforcement layer for the purely functional boundary.
7. **Cast** — Target-faithful projection. Same meaning rendered into whatever form the audience needs (model context, human summary, API response, print).
8. **Boot** — Minimal capsule for cold-start hydration. Deterministic, versioned, compressed. Same boot card + same model = same starting posture.
9. **Store** — Durable, retrievable persistence. Not a bin — the condition that makes everything else possible.
10. **Lore** — (Provisional) Accumulated narrative coherence that emerges over time. May be composite of other primitives.

### The Card Protocol (the formalization)

Six canonical cards define the protocol:
- **Spec Card** (v1.0.1) — Self-hosting definition of what a card is
- **Schema Card** — JSON Schema validator
- **Boot Card** — Genesis/cold-start
- **Sieve Card** — Shaping/compression
- **Cast Card** — Host projection
- **Check Card** — Independent verification

Each card is a small JSON artifact built to survive handoff. Key invariants:
- Cards preserve reconstructable capability (anti-pyramid law)
- Card ≠ Event (separate objects linked by reference)
- No card becomes canonical without explicit ratification
- Invalid cards cannot mutate canonical state (fail-closed)
- Fenton's Law: at every compaction boundary, emit enough deterministic state for cold continuation

### The Trust Lattice

Claims have binding tiers:
- `observed` — extracted, not binding
- `proposed` — corroborated enough to test
- `ratified` — explicitly human-ratified

Promotion is one-way and gated. The human gate is the product.

The undeniable value pitch: production AI systems are failing on these TODAY. Commercial attack toolkits exist. OWASP has them at #1. The substrate's pure function boundaries with hash-chained provenance are a direct architectural response.

### Chain of Custody Problem (Key Insight)

The pure function is honest. The BOUNDARIES around it are where trust breaks:
1. Input could be tampered before entering
2. A different function could be substituted
3. Output could be swapped after

Solution: hash chain where each step includes the previous step's hash as provenance. Check independently re-runs the pure function and compares. **This is why check exists as a separate primitive.**
